# SSH Tunnels ‚Äî Reference

## How It Works

1. Ed25519 keys are generated on the **host** and bind-mounted read-only into the container.
2. `entrypoint.sh` generates SSH config from `config.json` and pre-seeds `known_hosts` (first boot only ‚Äî skipped if entries already exist).
3. The agent SSHes to the target host, landing in a workspace session with access only to configured `project_paths`.
4. Per-host `forward_ports` in `config.json` control which ports the agent may tunnel to itself via `PermitOpen` on the remote sshd.

---

## Setup (quick reference)

```bash
make workspace-setup HOST=my-host     # workspace + SSH key + sshd reload
make test HOST=my-host                # verify: should print "dev-bot"
```

For remote hosts:
```bash
make workspace-setup HOST=my-host REMOTE_KEY=/path/to/key [REMOTE_USER=ubuntu]
```

After re-keying or workspace teardown:
```bash
make sync HOST=my-host      # re-sync SSH key (alias for key-add)
```

## Workspace Layout

### `container` isolation

The agent session is routed via `ForceCommand` into a long-lived rootless-Docker
container. Project directories are bind-mounted into the container:

```
workspace container (rootless Docker, dev-bot uid)
‚îî‚îÄ‚îÄ ~/workspace/
    ‚îú‚îÄ‚îÄ project1/    ‚Üê bind mount of /path/to/project1
    ‚îî‚îÄ‚îÄ project2/    ‚Üê bind mount of /path/to/project2
```

SSH/network-pivot binaries are stripped from the workspace image:
`ssh`, `scp`, `sftp`, `ssh-keygen`, `ssh-keyscan`, `ssh-agent`, `ssh-add`,
`nc`, `netcat`, `ncat`, `telnet`, `rsh`, `rlogin`, `socat`

### `restricted_key` isolation

No workspace container. The agent SSHes as `dev-bot` with a restricted key.
Project access is controlled by `project_access` in `config.json`:

- `acl` (default): POSIX ACLs on source paths; symlinks created in `~/workspace/`
- `copy`: files rsynced into `~/workspace/` under dev-bot ownership

## sshd Match Block (`container` mode)

Generated by `workspace_up.sh` for each host. Always replaced on re-run.

```
# BEGIN Dev-Agent Workspace Configuration: <host>
Match User dev-bot
    X11Forwarding no
    AllowTcpForwarding local          # "no" if forward_ports is empty
    PermitTunnel no
    PermitOpen localhost:3000 ...     # omitted if forward_ports is empty
    ForceCommand /usr/local/bin/openclaw-session-entry workspace-<host>
# END Dev-Agent Workspace Configuration: <host>
```

## Troubleshooting

### SSH falls back to password prompt
Key mismatch or missing `authorized_keys`.
```bash
make sync HOST=my-host
```

### Permission denied (publickey)
```bash
make workspace-setup HOST=my-host    # re-run workspace setup
make key-add HOST=my-host
```

### Host key changed error
```bash
ssh-keyscan -H <hostname> >> .ssh/known_hosts   # pre-seed the new key
make restart AGENT=<agent>
```

### SSH command not found in workspace
Expected ‚Äî SSH tools are intentionally excluded from the workspace image for security.

## Script Reference

| Make target | Raw script | What it does |
|---|---|---|
| `make keys` | `scripts/ssh_key/init_keys.sh` | Generate Ed25519 keys on host |
| `make workspace-setup HOST=<h>` | `workspace_up.sh` + `add.sh` + sshd reload | Full host onboarding |
| `make workspace-setup HOST=<h> REMOTE_KEY=<k>` | `scripts/remote/setup.sh` | Remote host onboarding |
| `make key-add HOST=<h>` | `scripts/ssh_key/add.sh` | Install public key on host |
| `make workspace-clean HOST=<h>` | `workspace_down.sh` + `remove.sh` + sshd reload | Tear down workspace |
| `make workspace-purge HOST=<h>` | `workspace_down.sh --purge` + `remove.sh` | Full teardown, removes dev-bot user |
| `make firewall` | `scripts/firewall/setup_firewall.sh` | Apply egress iptables rules |
| `make auth` | `scripts/nginx/init_htpasswd.sh` | Generate dashboard `.htpasswd` |
