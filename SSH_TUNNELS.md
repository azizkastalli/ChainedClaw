# SSH Tunnels — Reference

## How It Works

1. Ed25519 keys are generated on the **host** and bind-mounted read-only into the container.
2. `entrypoint.sh` generates SSH config from `config.json` and pre-seeds `known_hosts` (first boot only — skipped if entries already exist).
3. The agent SSHes to the target host, landing in a chroot jail with access only to configured `project_paths`.
4. Per-host `forward_ports` in `config.json` control which ports the agent may tunnel to itself via `PermitOpen` on the remote sshd.

---

## Setup (quick reference)

```bash
make setup HOST=my-host     # chroot + SSH key + sshd reload
make test HOST=my-host      # verify: should print "dev-bot"
```

For remote hosts:
```bash
make remote-setup HOST=my-host REMOTE_KEY=/path/to/key [REMOTE_USER=ubuntu]
```

After re-keying or chroot teardown:
```bash
make sync HOST=my-host      # re-sync SSH key (alias for key-add)
```

## Chroot Layout

```
/srv/chroot/dev-bot/
├── bin/, lib/, lib64/          ro bind from host
├── usr/lib/, usr/lib64/        ro bind from host
├── usr/bin/                    symlinks (SSH/network tools excluded)
├── proc/, sys/                 dedicated ro filesystems
├── dev/                        null, zero, pts only
├── etc/                        minimal passwd + group
└── home/dev-bot/
    ├── project1/               rw bind → /path/to/project1
    └── project2/               rw bind → /path/to/project2
```

**Excluded from chroot** (symlinks not created):
`ssh`, `scp`, `sftp`, `ssh-keygen`, `ssh-keyscan`, `ssh-agent`, `ssh-add`,
`nc`, `netcat`, `ncat`, `telnet`, `rsh`, `rlogin`

## sshd Match Block

Generated by `jail_set.sh` for each host. Always replaced on re-run.

```
Match User dev-bot
    ChrootDirectory /srv/chroot/dev-bot
    X11Forwarding no
    AllowTcpForwarding local          # "no" if forward_ports is empty
    PermitTunnel no
    PermitOpen localhost:3000 ...     # omitted if forward_ports is empty
```

## Troubleshooting

### SSH falls back to password prompt
Key mismatch or missing `authorized_keys`.
```bash
make sync HOST=my-host
```

### Permission denied (publickey)
```bash
# Missing user in chroot /etc/passwd or wrong permissions
make chroot HOST=my-host    # re-run chroot setup
make key-add HOST=my-host
```

### Host key changed error
```bash
ssh-keyscan -H <hostname> >> .ssh/known_hosts   # pre-seed the new key
make restart
```

### SSH command not found in chroot
Expected — SSH tools are intentionally excluded for security.

## Script Reference

| Make target | Raw script | What it does |
|---|---|---|
| `make keys` | `scripts/ssh_key/init_keys.sh` | Generate Ed25519 keys on host |
| `make setup HOST=<h>` | `jail_set.sh` + `add.sh` + sshd reload | Full host onboarding |
| `make key-add HOST=<h>` | `scripts/ssh_key/add.sh` | Install public key on host |
| `make chroot-clean HOST=<h>` | `scripts/chroot_jail/jail_break.sh` | Tear down chroot |
| `make firewall` | `scripts/firewall/setup_firewall.sh` | Apply egress iptables rules |
| `make auth` | `scripts/nginx/init_htpasswd.sh` | Generate dashboard `.htpasswd` |
